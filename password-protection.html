Step by Step

How to set up Password Protection

From install to active protection in under 2 minutes.

1

Install & activate the plugin

Go to Plugins → Add New, search for "Smart Password Protect", click Install Now then Activate. A new menu item will appear in your WordPress admin sidebar.

2

Open Password Protection settings

Click Smart Password Protect in the admin menu. From the top navigation click Password Protection. You will see three sub-tabs: General, IP Addresses, and Permissions.

3

Set a site password

Under the General tab, enter your desired password in the Password field and click/tab out of the field — it auto-saves. The password is stored as a hashed value using WordPress's built-in wp_hash_password().

4

Enable protection

Toggle Enable Password Protection to ON. The toggle is disabled until a password is set. Once enabled, every page visit from an unauthenticated user will show the password form. A green ACTIVE badge appears in the Dashboard.

5

Configure permissions & IP whitelist (optional)

Use the Permissions tab to choose who can bypass protection (admins, logged-in users, REST API, RSS, login page). Use the IP Addresses tab to whitelist trusted IPs — a one-click "Auto Fill" button adds your current IP instantly.

6

Share the password with your team

Share the site password with whoever needs access. They enter it on the password form, check "Remember Me", and stay logged in for up to 7 days without re-entering the password. To deactivate, simply toggle protection off or clear the password field.

Frontend Experience

What visitors see when protection is active

When a visitor tries to access any page on your protected site, they are immediately shown a clean, responsive password form — no trace of your content is visible. The form features a password input, an optional "Remember Me" checkbox, and a submit button styled to match your brand.

  • Displays on every page, post, feed, and login page (configurable)
  • Wrong password shows an animated shake error message
  • Fully responsive — works on mobile, tablet, and desktop
  • "Remember Me" lets visitors stay authenticated for up to 7 days
  • Automatic noindex header sent while protection is active

Remember Me (Days) Default: 7

Number of days a visitor stays authenticated after entering the password and checking "Remember Me". If the visitor does not check Remember Me, they are authenticated for only 1 day regardless of this setting. Set to 1 for tightest security.

Permissions Tab

Protected Permissions — who can bypass?

Configure exactly which users, roles, and request types can access the site without entering the password. All settings auto-save on toggle.

Allow Administrators Default: ON Recommended ON

When ON, logged-in WordPress administrators (users with manage_options capability) bypass the password form automatically. Keep this ON or you will lock yourself out and need to disable the plugin via FTP/cPanel to regain access.

Allow Logged In Users Default: OFF

When ON, any WordPress user who is logged in (editors, authors, subscribers, etc.) can access the site without the password. Useful when you want to protect the site from the public but let your entire team through without sharing a separate password.

Allow RSS Feeds Default: OFF

When ON, RSS/Atom feed URLs (e.g. /feed/) are accessible without a password. Useful if you want to protect the site visually but still allow RSS readers and podcast apps to pull your content feed.

Allow REST API Default: OFF

When OFF, all unauthenticated REST API requests (/wp-json/) return a 401 rest_forbidden error. Enable this if your headless frontend or third-party service needs API access while the site is protected.

⚠️ Caution: Enabling REST API bypass may expose your content to unauthenticated API consumers. Only enable if your use case requires it.
Allow Login Page Default: OFF

When ON, the WordPress login page (wp-login.php) is accessible without entering the site password. Enable this when you want team members to log in to WordPress normally without needing the site-wide password.

IP Addresses Tab

IP Whitelisting — auto-bypass for trusted IPs

Add individual IP addresses to the whitelist. Any visitor whose IP matches an entry will bypass the password form completely — no cookie, no prompt, just transparent access.

1

See your current IP

The settings page detects and displays your current public IP address at the top of the section automatically.

2

Auto-fill your IP

Click "Auto Fill Input" and your current IP is pre-filled in the input field. Or click "Copy IP" to copy it to your clipboard.

3

Add IP & save

Type or paste an IPv4 address (e.g. 192.168.1.100) and click "Add IP". It saves instantly. Invalid format or duplicates show an inline error.

4

Remove IPs

Each whitelisted IP shows a "Remove" button in the table. Click it to instantly remove that IP from the whitelist.

⚠️ Dynamic IPs: Most home internet connections use a dynamic IP that changes periodically. If you get locked out, check your current IP at checkip.amazonaws.com or ipify.org and re-add it.

FAQ

Common questions

I forgot the password and locked myself out. What do I do?
Make sure "Allow Administrators" is ON (it is by default). Log in to wp-login.php normally with your WordPress credentials. As an admin you bypass the password form automatically. Once inside, go to Smart Password Protect → Password Protection → General and set a new password.
Will Google index my site while it's protected?
No. The plugin automatically adds X-Robots-Tag: noindex HTTP headers whenever protection is active. Google and other crawlers that respect this header will not index the protected pages. Once you disable protection, the noindex headers are removed and your site is crawlable again.
Is the cookie secure? Can someone steal it?
The cookie stores an HMAC-SHA256 signature of the hashed password + expiry time, signed with wp_salt('auth'). The plaintext password and its hash are never stored in the cookie. The cookie is HttpOnly (inaccessible to JavaScript), and Secure on HTTPS sites. Changing your password immediately invalidates all existing cookies.
Does it protect the REST API?
Yes. By default REST API access is blocked for unauthenticated users when protection is active. Requests return 401 rest_forbidden. Logged-in users always bypass. You can allow unauthenticated REST access under Permissions → "Allow REST API".
Can I set multiple passwords for different users?
Currently the plugin supports a single site-wide password. Multiple password support is on the roadmap. In the meantime, use the Temporary Login feature to give individual users their own time-limited magic links that don't share the site password.
Does it work with WooCommerce or page builders?
Yes. The protection works at the WordPress template_redirect hook level, which runs before any page content is output. This means it intercepts all requests regardless of theme, page builder (Elementor, Divi, Beaver), or plugin (WooCommerce, BuddyPress, etc.).
