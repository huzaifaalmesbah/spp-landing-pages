Step by Step

How Temporary Login works

From generating a link to the recipient accessing your site — here's the full flow.

1

Enable the feature

Go to Smart Password Protect → Dashboard and toggle Temporary Login to ON. A confirmation modal will ask you to confirm. Once enabled the Temporary Login tab appears in the top navigation.

2

Configure default settings (optional)

Open Temporary Login → Settings to set a global default role, configure auto-cleanup for expired links, set log retention days, and manage capability restrictions for temporary users.

3

Generate a magic link

On the Temporary Login → Generate tab, choose an Expiration Duration (1 hour, 1 day, 7 days, or Custom Date). Click "+ Show Advanced Options" to also set a Max Login Limit and Role. Click "Generate Link".

4

Copy & share the link

The generated magic link appears in a readonly box below the form. Click "Copy" and send it to your developer, client, or teammate via email, Slack, or any messaging tool. The link looks like: yoursite.com/?sppwp_token=abc123…

5

Recipient clicks the link

The recipient opens the link in any browser. The plugin validates the token, creates (or reuses) a temporary WordPress user with the assigned role, and logs them in instantly — no username, no password prompt. They are redirected to the admin dashboard.

6

Link expires & auto-cleans

When the link reaches its expiry time (or usage limit), it stops working. The temporary WordPress user is automatically logged out. The plugin's WP Cron job runs daily to delete expired links and temporary user accounts based on your cleanup settings.

Custom Date
Pick any specific date from the date picker. Expires at end of that day.
Max Login Limit Advanced · Optional

Limit how many times this link can be used to log in. Leave blank or set to 0 for unlimited uses. Set to 1 for a single-use link — after one login, the link becomes inactive.

0 = Unlimited 1 = Single use 5 = 5 logins max
Role Advanced · Optional

The WordPress role assigned to the temporary user. Defaults to the role set under Settings → Default Role. Choose the minimum role needed for the task.

Administrator Full access — can manage everything including plugins, themes, users. Capability restrictions still apply (see Capabilities section).
Editor Can publish and manage all posts/pages and categories. Cannot manage plugins, themes, or users.
Author Can write, edit, and publish their own posts only.
Contributor Can write and edit posts but cannot publish — requires admin approval.
Subscriber Read-only access to the dashboard. Can only manage their own profile.

Settings Tab

Global Settings & Capabilities

Configure global defaults, auto-cleanup schedules, and exactly what temporary users can and cannot do.

General

Default Role Default: Administrator

The WordPress role pre-selected when generating a new link. You can override this per-link in the advanced options. Set to a lower role (e.g. Editor) to enforce a safer default.

Expired Links Auto-cleanup (Days) Default: 0

How many days after expiry before the link and temporary user account are permanently deleted by the daily WP Cron job. Set to 0 to delete immediately on the next cleanup run after expiry.

0 days
Delete immediately
7 days
Keep for 1 week
30 days
Keep for 1 month
Logs Auto-cleanup (Days) Default: 0

How many days to retain activity log entries. Set to 0 to keep logs forever (recommended for audit purposes). Higher values automatically prune old log entries to keep the database lean.

Capabilities

Control what temporary users can do — even if they're assigned the Administrator role. These settings are global and apply to every temporary login session.

Allow Plugin Actions Default: OFF Recommended OFF

When OFF, temporary users cannot install, activate, or deactivate plugins — even with Administrator role. Enable only if the temp user explicitly needs to manage plugins.

Allow Plugin Deletion Default: OFF Recommended OFF

When OFF, temporary users cannot delete plugins from the server. Critical safeguard — keep this OFF to prevent accidental or malicious plugin deletion.

Allow Theme Actions Default: OFF

When OFF, temporary users cannot install or switch themes. Enable if the temp user needs to install or change a theme during their session.

Allow Theme Deletion Default: OFF Recommended OFF

When OFF, temporary users cannot delete themes. Keep OFF to protect your theme files from accidental removal.

🛡️ Always enforced (cannot be disabled):
  • • Temporary users cannot deactivate or delete Smart Password Protect itself
  • • Temporary users cannot edit profiles or manage other users
  • • Temporary users are automatically logged out when their link expires
  • • Temporary user accounts are auto-deleted after the cleanup period
Log Tab

Temporary Login Activity Logs

Every temporary login session is tracked in detail. The Log tab gives you a complete audit trail of all access events for compliance and security review.

📅

Date & Time

Relative time shown (e.g. "3 hours ago") with a tooltip showing the exact timestamp on hover.

👤

User

The temporary WordPress username associated with the login token.

⚡

Action

The event type: login, logout, expired, limit_reached, etc. Color-coded for quick scanning.

📝

Details

Human-readable description of the event with any relevant context (e.g. browser user agent, redirect URL).

🌐

IP Address

The public IP address of the temporary user at the time of the event. Shown in monospace for easy reading.

💾 Log retention: Configure how long logs are kept under Settings → Logs Auto-cleanup (Days). Set to 0 to keep all logs indefinitely.
